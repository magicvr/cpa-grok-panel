业务设置

保存后立即作用于新的用量事件和降权任务，无需重启插件。

自动刷新
仅在页面可见且没有账号操作时按间隔同步。
轮询间隔（秒）
账号列表同步 accounts；设置页只同步 settings 和 meta。
批量操作并发数
浏览器同时执行的启停、降权、解除降权、设置优先级和安全删除任务数。
出站代理（批量重签）
与 CPA proxy-url 相同格式（如 http://127.0.0.1:10808）。重签由插件进程直连 auth.x.ai，不走 CPA 业务代理；留空则跟进程 CPA_GROK_OUTBOUND_PROXY / HTTPS_PROXY / HTTP_PROXY。
每日清零
按插件进程所在服务器的本地时区清空请求数和 Token 累计。
每日清零时间
插件启动时若已跨过当天时间点，会补执行一次。
连续失败阈值
hard streak 达到该值时进入 hard 降权。
Soft demotion
failure debt 达到 soft 阈值时进入观察降权档。
Soft 目标优先级
soft 与 half-open 观察档使用的 priority。
Soft debt 阈值
默认 2.0；hard 条件始终优先。
Hard debt 阈值
debt 达到该值时直接进入 hard 降权。
401/403 debt 加分
每次归因的 401/403 增加的 failure debt。
429 debt 加分
仅在“429 计入阈值路径”开启时生效。
成功 debt 衰减
成功请求扣减 debt，但不会直接清空历史。
429 计入阈值路径
启用后，连续 HTTP 429 达到阈值时请求降权。
5xx 计入阈值路径
启用后，连续 HTTP 500–599 达到阈值时请求降权。
降权目标优先级
自动或手动降权时写入账号文件的 priority。
默认恢复优先级
没有可靠 baseline 时，“解除降权”回写的 priority。
优先级冷却恢复
soft/hard 降权经过 6h → 12h → 24h 冷却后进入恢复流程。
冷却恢复跳过机器人
开启后，显式机器人账号不参与自动冷却恢复；手动「解除降权」始终可用。默认开启。
Half-open 恢复
冷却到期先进入 soft 观察档，不直接恢复 baseline。
Half-open 成功阈值
观察档累计成功达到该值后恢复 baseline。
Free 用户日限额（token）
套餐为 Free 或无官方限额时，用量进度条分母；默认 2000000。不改变已缓存的套餐类型。
