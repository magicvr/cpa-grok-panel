业务设置

保存后立即作用于新的用量事件与存活 priority 绑定，无需重启插件。

自动刷新
仅在页面可见且没有账号操作时按间隔同步。
轮询间隔（秒）
账号列表同步 accounts；设置页只同步 settings 和 meta。
批量操作并发数
浏览器同时执行的启停、设置优先级和安全删除任务数。
出站代理（批量重签）
与 CPA proxy-url 相同格式（如 http://127.0.0.1:10808）。重签由插件进程直连 auth.x.ai，不走 CPA 业务代理；留空则跟进程 CPA_GROK_OUTBOUND_PROXY / HTTPS_PROXY / HTTP_PROXY。
每日清零
按插件进程所在服务器的本地时区清空请求数和 Token 累计。
每日清零时间
插件启动时若已跨过当天时间点，会补执行一次。
测活积分阈值
debt ≥ 阈值时清零并自动测活（source=auto）。默认 2.0。不再使用连败阈值触发。
401/403 debt 加分
每次归因的 401/403 增加的 failure debt。
429 debt 加分
仅在「429 计入 debt」开启时生效。
成功 debt 衰减
成功请求扣减 debt；若存活≠正常则回血为正常并写 priority_live。
429 计入 debt
启用后，HTTP 429 增加 debt_fail_429。
5xx 计入 debt 路径
启用后，HTTP 500–599 可参与 debt 计数路径（权重同 401 未单独配置时不额外加分）。
正常 priority
live 状态写入的 priority，默认 0。
无效 priority
invalid（401）写入的 priority，默认 -50。
死号 priority
dead（403）写入的 priority，默认 -100。
限流 priority
throttled（429）写入的 priority，默认 -50。
未知 priority
从未测活 / 重签后清空时写入，默认 10。
异常 priority
error（其它错误）写入的 priority，默认 -50。
Free 用户日限额（token）
套餐为 Free 或无官方限额时，用量进度条分母；默认 2000000。不改变已缓存的套餐类型。
