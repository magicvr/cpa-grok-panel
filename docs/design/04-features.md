# 04. 功能规格

每项均说明触发条件、行为、边界和失败表现。所有写功能受 `write_mode`、host capability、角色和 revision 条件约束。

## 1. 账号列表

- **触发条件**：打开面板、手动刷新或操作完成后刷新。
- **行为**：从 CPA 获取 xAI OAuth 非秘密元数据，与插件 usage、检查结果、降权状态合并；支持分页、排序、过滤。
- **边界**：列表是短时投影；provider 自由文本不作为唯一筛选依据；不读取 auth 文件内容。
- **失败表现**：host 不可用时返回明确错误和上次成功快照时间；若展示缓存快照，整页标注 stale，写按钮禁用。

## 2. 真实 token 用量累计

- **触发条件**：CPA 调用 `usage.handle`。
- **行为**：按 `auth_file_id`、`event_id` 去重后累计真实字段，更新请求计数、最后活动时间和数据质量。
- **边界**：不估算缺失 token；不把本地上限当实际供应商额度；不跨插件迁移历史。
- **失败表现**：非法事件拒绝并计指标；state 不可写时向 CPA 返回可重试错误；无可信身份时不归到账号且不降权。

## 3. 失败自动降权

- **触发条件**：某账号连续、可信、auth 可归责失败达到配置阈值，且 `write_mode=managed`。
- **行为**：保存当前 priority 基线，以条件写调用 host 将该 auth 优先级设为 `-100`，记录审计。
- **边界**：只以实际 `auth_file_id` 归因；成功、取消、全局错误按规则处理；不禁用或删除账号；不替代 CPA 调度。
- **失败表现**：能力缺失时显示 unsupported；revision 冲突时不覆盖并标记 superseded/conflict；host 失败时保留失败操作供查看，不伪称已降权。

## 4. 人工健康检查

- **触发条件**：operator 选择一个或多个账号点击“健康检查”。
- **行为**：创建 operation，通过 host 指定 auth 执行最小、只读、低成本探测；记录状态、时间、耗时和脱敏错误。
- **边界**：不定时执行；不因 unhealthy 自动删除；必须确认实际使用身份与目标一致；并发受设置限制。
- **失败表现**：超时/网络故障默认 `unknown`；认证明确失败可为 `unhealthy`；每项独立返回，其他项继续。

## 5. 人工套餐核实

- **触发条件**：operator 显式选择账号并确认可能产生外部请求。
- **行为**：调用经确认的套餐查询/观察能力，将正式字段映射到 TierObservation。
- **边界**：不按 token 或模型可用性猜套餐；结果有时间戳且可能过期；默认无自动核验。
- **失败表现**：未知响应映射为 `unknown` 并记录稳定证据码；不展示敏感响应正文。

## 6. Responses 实测

- **触发条件**：operator 对选定账号发起，确认会产生一次真实模型请求。
- **行为**：使用固定安全测试模板和严格输出上限，通过 host 锁定 auth 身份执行；展示成功、延迟、模型、真实 usage 和脱敏摘要。
- **边界**：不允许任意系统提示或无限输出；不保存完整响应；测试产生的真实 usage 仍通过正常 usage 回调计量，应用服务不得重复累计。
- **失败表现**：实际使用账号不一致则结果作废；超时可取消；外部错误逐项返回，不自动触发套餐修改。是否计入失败降权完全由 CPA 正常 usage 事件和归责规则决定。

## 7. 批量启用

- **触发条件**：operator 勾选账号并确认。
- **行为**：按有界并发逐项调用 `host.auth.set_enabled(true)`，返回 before/after 与新 revision。
- **边界**：不恢复 priority；已启用项为幂等成功/跳过；墓碑账号拒绝。
- **失败表现**：部分失败返回 `partially_succeeded`；revision 冲突要求刷新，不回滚已成功的其他账号。

## 8. 批量停用

- **触发条件**：operator 勾选账号并确认影响。
- **行为**：逐项条件写停用，保留账号和统计。
- **边界**：不删除、不改变 priority；保护规则可阻止停用最后一个可用账号（是否启用此规则待确认）。
- **失败表现**：与批量启用相同；UI 保持逐项结果可见。

## 9. 恢复优先级

- **触发条件**：operator 选择由本插件成功降权且存在 baseline 的账号。
- **行为**：确认 host 当前 priority 仍为降权值后，条件写恢复 baseline，降权状态置为 none/已恢复并审计。
- **边界**：不自动启用账号；没有 baseline 不猜值；外部已修改则不覆盖。
- **失败表现**：current priority 不符返回 `revision_conflict`/`priority_superseded`；批量时逐项处理。

## 10. 安全清理无效账号

- **触发条件**：admin 从单账号危险区发起预检，或对小批量账号逐项确认；提供精确文件名。
- **行为**：两阶段预检/确认，重新读取 host 状态，执行保护档位规则，以短期令牌绑定 id、文件名、revision 和 principal，最终调用精确删除。
- **边界**：必须有 CPA 管理鉴权；禁止通配符、路径、模糊匹配；不自动删除；不绕过 host API；建议 MVP 只支持单账号删除。
- **失败表现**：任何身份变化、保护命中、令牌过期、能力缺失均安全拒绝；删除结果不确定时标记 `unknown_outcome` 并刷新 host，不自动重试。

## 11. 设置管理

- **触发条件**：admin 打开设置页并提交完整表单。
- **行为**：字段校验、revision 条件更新、原子持久化，再更新 worker pool 等运行时配置。
- **边界**：只含并发、失败阈值、保护档位、token capacity、展示过期和超时等；不提供自动检查/自动删除开关。
- **失败表现**：字段级错误不改变任何值；revision 冲突要求刷新；运行时应用失败则回滚到旧设置并报告。

## 12. token 容量展示

- **触发条件**：账号列表/详情加载，或 admin 设置默认/单账号容量。
- **行为**：以真实 `totalTokens` 除以配置容量显示进度和超量状态。
- **边界**：容量是人为分母，不是 CPA 调度限制，不触发自动停用、降权或删除；缺少真实 total 时标注数据不完整。
- **失败表现**：无容量显示“未设置”；容量非法拒绝保存；计数超过容量仍显示实际比例，可大于 100%。

## 13. 操作审计与状态

- **触发条件**：任何人工检查、写操作、设置变更、自动降权。
- **行为**：创建 operation/audit，UI 可查看逐项状态；敏感内容脱敏。
- **边界**：审计不是通用日志替代；保留期可配置是后续范围。
- **失败表现**：若关键审计无法持久化，危险写操作应 fail closed；只读检查可按策略失败并告警，MVP 建议同样拒绝启动。

## 14. 明确禁止的自动化

系统不创建健康/套餐定时器，不依据 token 容量自动动作，不自动删除，不在降权后自动恢复。未来若需求改变，必须新增 ADR、安全模型和显式默认关闭设置，而非复用现有手动端点暗中调度。
